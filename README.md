# Minecraft Google Drive Backup Companion

Host-run backup companion for `mc-paper-velocity-geyser-stack`.

This project is meant to live beside the main Minecraft runtime on the Ubuntu VM and provides:

- app-consistent offsite `restic` snapshots to Google Drive through `rclone`
- nightly local archive backups by invoking the core stack's `backup-worlds.sh`
- staged restores from either local tar archives or offsite `restic` snapshots
- safe rollback promotion that preserves live player inventory by default

## Layout

- `scripts/offsite-backup.sh`: 30-minute encrypted snapshot workflow
- `scripts/nightly-local-backup.sh`: nightly local archive backup
- `scripts/restore-to-staging.sh`: restore a snapshot or archive into `staging/`
- `scripts/promote-rollback.sh`: promote a staged restore to live data
- `scripts/list-snapshots.sh`: inspect offsite snapshot history
- `scripts/backup-verify.sh`: daily restore verification helper
- `config/extra-paths.txt`: optional extra persistent paths to include in offsite snapshots
- `hooks/pre-backup.d/`: optional pre-backup dump hooks
- `ops/systemd/`: example service and timer units

## Quick Setup

1. Copy `.env.example` to `.env` and set:
   - `TARGET_STACK_ROOT`
   - `RESTIC_REPOSITORY`
   - `RESTIC_PASSWORD_FILE`
   - `GOOGLE_DRIVE_REMOTE`
2. Configure `rclone` on the Ubuntu VM for Google Drive.
3. Install `restic`, `rclone`, `rsync`, `tar`, and Docker CLI access on the VM.
4. Put the `restic` password in the configured password file.
5. Run:

```bash
chmod +x scripts/*.sh scripts/lib/*.sh
./scripts/offsite-backup.sh
./scripts/nightly-local-backup.sh
./scripts/list-snapshots.sh
```

## Safe Rollback

Restore to staging first:

```bash
./scripts/restore-to-staging.sh latest survival
```

Then promote that staged restore back to live while preserving current player inventory:

```bash
./scripts/promote-rollback.sh staging/restore-... survival
```

To intentionally roll back inventory too, pass:

```bash
./scripts/promote-rollback.sh staging/restore-... survival --allow-playerdata-rollback
```

## Systemd

Use the example units in `ops/systemd/`. Update the working directory path from `/opt/minecraft-backup-google-drive` if needed before installing them under `/etc/systemd/system/`.

## Public Repo Safety

This repo is intended to be safe for public GitHub hosting, but live secrets and runtime artifacts must stay local.

Do not commit:

- `.env` or any other environment override file with real values
- `secrets/` contents such as the `restic` password file
- `rclone.conf` or any OAuth/token material
- `logs/`, `runtime/`, or `staging/` contents generated by backup or restore runs
- any hook output or extra-path target that contains private database dumps

Before publishing, it is worth one last `git status --ignored` check from the repo root to confirm only intended files are tracked.
